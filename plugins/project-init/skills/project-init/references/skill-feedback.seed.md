# skill-feedback — 스킬·에이전트 마찰 포착 (day 1부터)

이 프로젝트에서 스킬·에이전트가 기대만큼 못 할 때, **판단하지 말고 한 줄만 append**한다.
포착은 공짜여야 한다 — 작업 흐름을 끊지 않는 게 포착률의 전부. 일반적인지 여부는 나중에 따진다.

```
- [스킬·에이전트명] 무슨 마찰 — 최소 재현 예시 / 기대 vs 실제
```

규칙:
- **그 자리에서 일반성 판단·수정 금지.** 기록만 한다.
- 고치려 들지 않는다 — 흐름 유지가 우선.
- 설치된 스킬 파일을 직접 고치지 않는다 — 업데이트 때 날아간다. **프로젝트는 소비자다.**
- 프로젝트가 끝나면 이 로그를 정본 강화 루프로 넘겨 승격(일반성·재현성·회귀·삭제우선) 여부를 따진다.

> 핵심 비대칭: **포착은 싸게, 승격은 비싸게.** 포착이 비싸면 아무것도 안 잡히고,
> 승격이 싸면 정본이 오염되고 비대해진다.
