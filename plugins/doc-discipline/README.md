# doc-discipline

문서를 **만드는 규율** 3종 번들. 무엇을 쓰느냐(문안·디자인)가 아니라, 어떤 구조로 내고
어떤 순서로 고치고 누가 완료를 선언하느냐를 고정한다. 렌더링·계측 자체는 이 번들 소관이
아니다 — `slide-craft`(읽는 지면)·`deck-harness`(발표 덱)가 소유한다.

## 설치

```
/plugin marketplace add sungwooHa/skills-marketplace
/plugin install doc-discipline@skills-marketplace
```

의존성 없음 (산문 규약 3종 · 실행 스크립트 없음).

## 스킬 3종

| 스킬 | 무엇을 고정하나 | 언제 발동 |
|---|---|---|
| `dual-altitude-brief` | 한 사안 = **두 지면의 짝**(고맥락 ↔ 저맥락) + 항목 번호 상호 참조. **짝 없는 항목 = 미완** | 승인도 받아야 하고 구현도 해야 하는 사안 · 보고서와 설계서가 갈라질 때 |
| `ssot-doc-pipeline` | **원고(SSOT) → 맥락(제작 스펙, 문서당 1개) → 산출물(파생·직접 수정 금지)** 3층 + 수정 순서·드리프트 점검·동결·승격 | 산출물로 렌더되는 문서 · 세션을 넘겨 반복 수정되는 문서 |
| `produce-verify-pair` | 제작↔검증 **페르소나·세션 분리**, 검증자는 판정만·수정 금지, PASS/FAIL 이분 + default-deny, 종료 조건 | 검증을 붙일지 말지 판단할 때 · "에이전트끼리 토론시켜" 요청이 들어올 때 |

세 스킬은 서로를 참조한다 — 짝 지면은 파이프라인의 산출물이고, 파이프라인의 승격 조건이
검증 PASS다. 그래서 별개 플러그인이 아니라 한 번들이다.

## 왜 이 셋인가 (해결하는 실패 3종)

- **비유만 남은 승인** — 은유로 명세를 대신하면 승인은 나는데 구현은 시작되지 않는다.
  반대로 스펙만 있으면 전달에 실패한다. `dual-altitude-brief`가 둘을 한 배송 단위로 묶는다.
- **산출물 직접 수정 드리프트** — LLM이 가장 흔히 저지르는 사고. 결함이 눈에 보인 곳(HTML)에서
  고치고, 원고가 뒤처지고, 다음 재생성 때 수정분이 조용히 날아간다. 3층 + 수정 순서로 구조적 차단.
- **자기 검증과 왕복 남용** — 만든 세션이 스스로 "검증 완료"를 선언하면 느슨해진다. 동시에
  에이전트끼리 자율 왕복 토론은 토큰만 태우고 같은 오해로 수렴한다. 왕복이 값을 하는 자리는
  **제작↔검증 적대적 루프 하나뿐**이라고 못 박는다.

## 기존 플러그인과의 경계

| | 소관 | 이 번들 |
|---|---|---|
| `deck-harness` | 발표 덱 제작·5축 검증관 실물 구현 | 짝을 **언제 쓰지 말아야 하는지**와 왕복 금지 원칙 (deck-harness에 없는 부분) |
| `slide-craft` | 읽는 지면 저작 + 정량 게이트 스크립트 | 계측을 누가 판정하고 언제 승격하는지 |
| `project-init` | 프로젝트 뼈대·에이전트 템플릿 생성 | 생성된 프로젝트가 문서를 굴리는 방식 |

`produce-verify-pair`는 deck-harness의 검증관 기계를 **복제하지 않는다** — 상호참조만 건다.

## 출처

AX기반개발TF 프로젝트의 운영 에이전트(총괄 2종·제작관 2종·검증관 1종)에서 **패턴만** 증류했다.
원본 에이전트는 프로젝트 경로·도메인 어휘에 결합돼 있어 그대로 승격하지 않았다.

| 패턴 | 출처 에이전트 | 걷어낸 결합 |
|---|---|---|
| 고맥락↔저맥락 짝 지면 | `chief-architect` · `program-director` 공통 §산출 규격 | 도시 은유·회장 언어 체계·실명 인물 맵 → 라벨 붙인 범용 예시로 치환 |
| 3층 위계 파이프라인 | `report-producer` · `slide-producer` + 프로젝트 CLAUDE.md 파일 규약 | 산출 경로 5종 하드코딩 → `<docs-root>` 추상 지칭 |
| 제작↔검증 적대적 짝 | `slide-qa-auditor` 철칙 5종 + 프로젝트 협업 원칙 | 기준 문서 ID 체계·폐기 용어 목록·계측 스크립트 경로 → 소비 프로젝트 소유로 외부화 |
